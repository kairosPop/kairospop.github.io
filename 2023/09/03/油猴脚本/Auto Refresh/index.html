自动刷新网站 - Auto Refresh

需求：写一个自动刷新网站的油猴脚本

这个脚本会在打开 https://www.example.com/ 网站时自动刷新页面。在脚本中设置了刷新间隔为 5 秒（timeInterval = 5000），也就是每隔 5 秒刷新一次页面。你可以根据需要调整刷新间隔值，以适合你的使用需求。

代码刨析：

当油猴脚本被激活并应用于匹配 https://www.example.com/* 的网站时，脚本将执行以下操作：

  1. 定义一个匿名函数 (function() { ... })()，该函数会自动执行。
  2. 在函数中，我们使用 setTimeout 函数来设置一个定时器，该定时器在指定的时间间隔后触发刷新操作。
  3. setTimeout 函数接受两个参数：一个是要执行的函数或代码块，另一个是延迟时间（以毫秒为单位）。
  4. 在这个脚本中，我们将刷新操作 location.reload() 放在了定时器内部，以确保在每个时间间隔达到后都会触发页面刷新。
  5. 在示例代码中，我们将刷新间隔设置为 5 秒（timeInterval = 5000），即每隔 5 秒刷新一次页面。
  6. 最后注意到 @match 注释，它指定了该脚本适用的网站 URL 匹配模式。在这个例子中，脚本适用于以 https://www.example.com/ 开头的所有页面。

‘use strict’; 是什么？

'use strict'; 是 JavaScript 中的一个指令，它用于启用严格模式（strict mode）。严格模式是一种对 JavaScript 解析和执行的限制模式，它使得代码更加规范、可靠，并且有助于减少常见的错误。

使用 'use strict'; 的好处包括：

  1. 变量必须声明后再使用：在严格模式下，所有变量都必须使用 var、let 或 const 关键字进行声明，否则会抛出错误。
  2. 禁止使用未声明的变量：在严格模式下，使用未声明的变量会抛出错误。
  3. 禁止删除变量、函数和函数参数：在严格模式下，使用 delete 操作符删除变量、函数、函数参数会抛出错误。
  4. 函数内部的 this 值为 undefined：在严格模式下，非严格模式中函数内部的 this 默认绑定到全局对象（如 window），而在严格模式下，则是 undefined。
  5. 不允许重复的函数参数名：在严格模式下，函数定义中出现重复的参数名会抛出错误。

通过在脚本中添加 'use strict';，可以确保代码在更严格的环境下执行，提高代码的可靠性和可维护性。
